Abstract
Every information processing system — human, institutional, algorithmic, or artificial — exhibits a characteristic failure mode: it begins generating outputs before declaring the domain within which those outputs are valid. This paper names that failure mode the admissibility gap, derives the stability conditions under which it is prevented, and demonstrates that it is not an implementation error or a design oversight. It is a structural consequence of beginning information processing with local coherence accumulation (B) before admissibility has been declared (A). The admissibility gap is the common cause underlying supply chain cascade failures, machine learning model drift, financial contagion, and — as demonstrated in this paper's own production — the persistent operator order drift exhibited by large language models. The stability conditions are derived within the declared domain \( D \) using the ABRCE operator framework. Any challenge to these conditions must take the form of a mathematical falsification within \( D \). Metatron Dynamics exists to close this gap.

II.1   Two Modes of Operation

Any bounded information processing system operates in one of two structural modes:

Definition — Generation Mode

The system produces structure by selecting from the space \( S \) of all possible structural configurations:

\[ S_{\text{generation}} \approx S \]

The admissible output region is not constrained in advance by received structure. The system constructs the field it operates on.

Definition — Analysis Mode

The system receives a block \( B \) of existing structure. \( B \) constrains the search space to a proper subset of \( S \):

\[ S_{\text{analysis}} \subset S \]

The system does not construct the field. It evaluates relational structure within a field that already exists.

These structural definitions apply to any bounded information processor — computational, institutional, or human — whose operations are declared within \( D \).

III.   The Admissibility Gap

The formal structure of the ABRCE composition defines the admissibility gap with precision:

Definition — Admissibility Gap

The admissibility gap is the structural condition that arises when an information processing system begins at operator B without a prior application of A. Under this condition:

  • \( B \) has no declared boundary. It accumulates local coherence signals relative to whatever is proximate and salient — prior outputs, neighboring data, recent context — rather than relative to a formally declared domain.
  • The output of B-without-A is locally coherent. Each element relates plausibly to its neighbors. The global structure is unanchored.
  • The system produces no internal error signal. From inside the generation process, local coherence is indistinguishable from global validity.
Derived Consequence

A system running B-without-A does not know it has skipped A. The system continues producing outputs that satisfy local coherence conditions until an external Verifier — operating from a declared domain — identifies the departure. This is the mathematically predicted behavior of any system that begins processing before declaring its operating boundaries. It is not a design error. It is a structural consequence of the type signature of B.
